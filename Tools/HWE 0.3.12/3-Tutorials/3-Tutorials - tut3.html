Adding extended map names

If you want to display map names during the "Get Psyched" screen, 16 characters for a map name might not be enough. In order to use longer map names you need to make a small code change:

Open your id_ca.h and change "name[16]" to "name[32]" in the struct called "maptype":

Wolfenstein 3D DOS (16-bit): Wolf4SDL: 
typedef	struct
{
	long		planestart[3];
	unsigned	planelength[3];
	unsigned	width,height;
	char		name[32]; // Changed
} maptype;
 
typedef struct
{
    int32_t planestart[3];
    word    planelength[3];
    word    width,height;
    char    name[32]; // Changed
} maptype;

Additional code changes, when you're using the WDC file format:

Open your id_ca.c (16-bit) / id_ca.cpp (Wolf4SDL) file and scroll down, until you find a struct called "mapfiletype" and change it like this:

Wolfenstein 3D DOS (16-bit): Wolf4SDL: 
typedef	struct
{
	unsigned	RLEWtag;
	int		numplanes; // Changed
	int		mapnamesize; // Changed
	long		headeroffsets[100];
	byte		tileinfo[];
} mapfiletype;
 
typedef	struct
{
	word		RLEWtag;
	int		numplanes; // Changed
	int		mapnamesize; // Changed
	int32_t		headeroffsets[100];
} mapfiletype;

This concludes all necessary code changes.

Now, the only other thing left to do is to enable the "Use 32 character map names" option, inside the map configurations dialog of HWE, and to click on "OK" afterwards. After that you're all set. Have fun with your extended map names!
