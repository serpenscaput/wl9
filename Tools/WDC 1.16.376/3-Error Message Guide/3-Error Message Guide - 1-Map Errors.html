Error Message Guide :: Map Errors

Error Description Resolution
Map has no player start position. The player needs to start somewhere.
Additional start position found at x,y. You only need one start position. Choose the correct one and get rid of the others.
Player start position is in a wall at x,y. The player shouldn't be inside of a wall.
Secret door at x,y is not on a wall. Secrets only work when placed on a wall.
Turning point at x,y should not be inside a wall. Turning points should be placed on floors.
Turning point at x,y should not be inside a door. Turning points should be placed on floors.
[objectname] at x,y should not be inside a wall. Objects should be placed on floors.
[objectname] at x,y should not be inside a door. Objects should be placed on floors.
Incomplete path for moving guard at x,y. Collision occured at cx,cy. Last turning point was at lx,ly. You need to set up some more turning points for the moving guard found at lx,ly.
The deaf guard at x,y is blocking a door. Deaf guards directly in front of doors can cause the door to be invisible and unopenable as well as just causing everything around there to act funny.
Map code 0 is used at x,y. Wall code 0 does not mean anything
Non-wall map code used on map border at x,y. This is not necessarily a huge error. Just check to make sure the player cannot leave the map.
Locked door requires key #n, which is not in the map. You have a locked door, but no key and no guard which drops that key.
Key #x does not unlock any doors. You have a key, but no door for that key.
Too many guards: n, max: 149. Unless you have made source code changes, you can only have 149 guards per map.
Too many static objects: n, max: 399. Unless you have made source code changes, you can only have 399 static objects per map.
Too many doors: n, max: 63 Unless you have made source code changes, you can only have 63 doors per map.
Map has no guards. Not having any guards used to cause "divide by 0" errors.
Map has no secrets. Not having any secrets used to cause "divide by 0" errors.
Map has no treasure. Not having any treasure used to cause "divide by 0" errors.
