Auto floor adds more than just floors... what's going on?
WDC relies heavily on how you define your map data codes. This probably means that that map code is defined as a floor instead of a wall.

To rememdy the situation: 1) Go to the "Edit" menu and select "Map Symbols."
2) Scroll until you find map code WDC mistakenly thinks is a floor.
3) Now look at the "Type". Is it defined as "Floor"?
4) If it is "Floor", set it to "Wall" or whatever it should be.

If it did not say "Floor", then there is a serious problem that needs to be addressed. Please let me know about it.


I'm using the Tile16s to view the Wolf3D Alpha, but the guards face the wrong directions. How can I fix this?
The Wolf3D Alpha is the only game that has a Tile16M for object code 0. To get the objects to match the correct map codes:
1) Go to the "File"menu and select "Project Information."
2) Check the box that says "Tile16Ms start at map code 0."
3) Click "OK" and the map tiles will be reloaded.

How do I get out of READ ONLY mode?
You need to set up an output folder in the project information screen. This folder needs to be different from your base data folder. Keeping things separate like this helps insure that you won't lose any data.
