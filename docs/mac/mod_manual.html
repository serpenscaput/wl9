SIMPLE USER CONFIGURATION KIT (S.U.C.K.)

Suck-ing is a super-easy method to modify the game and barely an inconvenience. Even your brother's dog can do it! It's limited in scope, not overly complex and has some loose screws but it lets you do some quick changes. With enough creativity it gives you plenty of room to turn the whole game upside down. All you need is a text file, add some variables from below and rename the file to mod.cfg. It allows to tweak game variables per level. You could have weapons and enemies behave totally normal in one level and then absolutely crazy in the next.

To sum up the 4 steps to modding suckcess:
S - Set up a text file
U - Unleash the variables
C - Call the file ????
K - Killing Profit

GAMEDICT.WL6
 File contains various options to configure your levels and is also described in detail in the original documentation. It gives you access to following features:

  • Floor and Ceiling Colors
    • Default is the color for any level not specified.

    Level Color
    ceilcolor 2 66
    floorcolor 21 42
    floor_default 0
    ceil_default 233

  • Floor and Ceiling Textures
    • Uses plane 3 to set textures per tile.

    texflats

  • Shading Color and Strength
    • Level Amount
    shading 1 5
    shadegreen 1 60
    shadered 1 0
    shadeblue 1 110
  • Parallax Sky
    • The texture number from the vswap and how many textures it should use.

    Level Num
    parallaxstart 1 60
    parallaxcount 1 16

  • Music Order
    • Music is in the music folder in ogg format. Custom names are supported. Titlemusic is for the intro of the game and intermusic for the intermission screen.

    Level Track
    levelmusic 21 "PLODDING"
    levelmusic 11 "my_new_music"
    titlemusic "epicsong"
    intermusic "BETWEEN"

  • Sound List
    • Option to rename the default sound files. First you specify the file in the gamedict. The entire sound order can be found in the source code (asmcref.h)

    sounds   "soundlist.txt"

    In SOUNDLIST.TXT:

    snd 0 "test.wav"
    snd 1 "player/test.wav"
    snd 2 "player/44.wav"
    snd 3 "guard/1.wav"

  • Episode/Scenario ending on a specific map
    • End and start each episode on a specific level number. Finalmap can be used multiple times.

    Episode Level
    startepisode 1 1
    startepisode 2 30
    finalmap 3
    finalmap 32

  • Episode Names
    • Scenario/Episode names for the menu. It got room for 20 episodes.

    Episode Name
    episode 1 "Escape from Castle Golfenstein"
    episode 2 "Operation: Eisenfrust"
    episode 3 "Attack of the Space Mutants"

  • Level Order
    • Beating the given level will take you to the specified map. Good for non-linear level flow in combination with the secret elevator and level loops.

    Level Next Level
    nextmap 1 5

  • Secret Level Order
    • Which level should lead to a secret map and back. This example takes you via the secret elevator from level 1 to 10. Beating the secret map will lead to level 2.

    Level Next Level
    secretlevel 1 10
    secretback 10 2

  • Par time
    • Partime for the given level.

    Level Seconds
    partime 1 105
    partime 14 442

  • Mirrored Wall Sides
    • Flips wall textures on the x-axis. Commonly used in mods for truck textures. 1 = horizontal, 2 = vertical, 3 = both sides.

    Wall id Flip-side
    mirwall 1 3
    mirwall 33 2

  • Darker Wall Sides
    • Darkened wall sides via the darktable. 333 = horizontal side, 666 = vertical side, 667 = both sides. See custom darktable in Various Things.

    Wall id Dark Side
    dwalls 0 (door sides) 667
    dwalls 11 333
    dwalls 23 666

  • Palettes
    • Color palettes used in the game. 1 = ingame, 2 = intermission, 3 = id logo, 4 = MacPlay logo, 5 = title screen, 6 = new game menu. The palettes are in the 768-bytes format used by ChaosEdit.

    Palette File
    palette 1 "wolfy.pal"
    palette 2 "Intermission_Pal.BRGR"

  • Level Number on Statusbar
    • Customized level number display.

    Level Display
    areascnum 1 99
    areascnum 2 74

  • Readme + Intermission Text + End Text
    • Option to have readme in the banana menu and text screens after specified levels. Leveltext displays it at start of the level and endtext at end of the final level.

    Level File
    readme "filename.xxx"
    leveltext 3 "folder/level5.art"
    leveltext 5 "story/helpart.wlt"
    endtext 7 story/end1.txt

  • Level-specific VSWAP
    • Option to pick vswap for specific levels. Allows to reuse vswaps without multiplying files which could be the case of the alternative method.

    Level File
    vswap 1 "file.ext"
    vswap 2 "vswap.wl3"

  • Tile Shading
    • Writing planelight in the config will enable the 4th (light) + 5th (teleporters) plane for the gamemaps and lets you adjust brightness for each tile. You pick one of the editors that supports more than 3 planes (WDC, HWE [save maps with WDC compatibility]), enable 5 planes and set the value for each tile on plane 4. Tile values range from 0 (bright) to 31 (dark). The other shading variables in the gamedict need to be set as well.

    planelight

  • Teleporters
    • First you specify the file in the gamedict. The game will enable the 5th plane which requires a suitable editor (WDC, HWE).

    teleporter   "my_tele.txt" // put in conf folder

    In MY_TELE.TXT:

    plane 5 id o(neway) : exclude p(layer) e(nemy) a(ll) : player angle
    teleport 1 "o:e:122"
    teleport 3 "b:e:0"
    teleport 4 "b:p"
    teleport 63 "o:e"

    The values (in this example: 1, 3, 4, 63) are placed on the 5th plane.
    Place teleport.wav (default) in the sound folder if a sound effect is desired.
    Each teleport is either paired (entry and exit or reverse) or a one-way ticket.
    1 goes to 2 and vice-versa.
    3 goes to 4
    5 and 6
    and so on
    teleport 1 "b:e:122" means id = 1, both ways, player only and angle will be set to 122 after the player teleports. The angle can be left out.

Level Specific Config
It is possible to have a config for each single level by adding the level number at end of the file name. If there is none for a specific level it will fall back on the main config.

modcfg "name"
Add this in GAMEDICT.WL6 to set the name for the mod config or else it uses the default file name "mod.cfg". Configs should be located either in "macenwolf/conf" or "macenwolf/modname/conf". The game checks the mod folder first if a mod is loaded (--file command) before checking the main folder. If there is no config file the game will use the default game settings.

For example in Gamedict.WL6:
modcfg   "funguns"

The game will look for a cfg file with the name "funguns.cfg", "funguns02.cfg" for level 2, "funguns37.cfg" for level 37 and so on. Default naming would be: mod.cfg, mod02.cfg, etc.

Actor Limit
The maximum per level for a mod is 5000 right now. So be a bit wary when spamming around enemies and projectiles in a level.

Auto-Level-Specific VSWAP
You can have a vswap for each single map by naming the file accordingly. Vswap02.wl6 would be for level 2, vswap44.wl6 would be for level 44. Alternatively vswap files can be also assigned via the gamedict.wl6.

Multiple Sound Folders
Same as the vswap you can have one for each level. Name the folder sounds02 for level 2 and so on. If it can't find a sound file in macenwolf/mod/sounds02 it will look for it in macenwolf/mod/sounds and afterwards go for macenwolf/sounds.

Optional level mission to end the map or the game (see finalmap definition in gamedict.wl6).

  • finditem: Collect a specific item (map id).

This would end the map as soon as you collect the gold key (map id = 43).

darktable "name.extension"
Shade effect for vertical/horizontal walls, which can be set manually in the gamedict.wl6 via "dwalls" or by using the 3rd plane. It's not an actual palette and just the numbers of your current palette shuffled around to remap the colors. If you set all values to 0 the color of the wall sides would turn into the first color of your palette. You could encode color specific messages on certain walls or have blood stains on walls being turned yellow.

darktable "light.pal"

vswap64
Writing simply vswap64 into your config will make the game use 64x64 textures and sprites. Prepare your vswap accordingly or the visuals will be all over the place. Add "64" at the end of your file name (myfile64.wl6).
File naming via the auto-method for individual levels as followed: vswap0264 for level 2, vswap8964 for level 89.

Wall Usage (_use)
Makes a wall interactive. Does one or multiple of the following if the player presses the use key.

"weapon:" - gives the player a weapon
"health:" - gives the player health
"armor:" - gives armor
"ammo:" - gives ammo
"gas:" - gives cans of gasoline
"rocket:" - gives missiles
"treasure:" - hands out gold
"modspr:x:y:s" - modifies a sprite at the given coordinates

wall_use   8   "ammo:100"
wall_use   8   "treasure:5"
wall_use   8   "ammo:25"
wall_use   8   "weapon:5"   // 5 = flamer, 6 = rockets

The modspr function either removes or changes sprites while making them traversable.
One wall can modify up to 50 sprites.
You could turn a gang of blocking npc sprites into traversable blood pools with one wall button.

wall_use   8   "modspr:09:07"   // will remove sprite at x=9 y=7
wall_use   8   "modspr:12:17:147"   // changes to sprite num 147 and makes it traversable
wall_use   8   "modspr:03:61:140"   // x=3, y=61, new spritenum = 140

Wall After Usage (_afteruse)
After pressing the use key the wall can be set to a new wall id.
For example you can jump to an empty shelf wall and make wall-8 only once usable
You could also chain them together and make wall-12 give something else.

wall_afteruse   8   12   // old + new wall id

Flat Animation
flat_frames   82   8   // floor/ceil texture id + frame amount
flat_animspeed   82   2   // flat id + animation speed

Flat Usage (_use)
Gain damage or health while standing on a floor/ceiling tile (plane 3).
Effect is once per second.
Also affects enemies and can go over their max HP.

flat_use   82   "damage:16"
flat_use   65   "health:20"

============================== STATIC OBJECTS ==============================

A simple method to modify existing objects. You know those things that are plastered all over a level, always block your way and do nothing else except stare at you when you come by every 5 minutes. Another way is to use the Custom Objects method to create new objects from scratch.

Solid Objects (_solid_)
Make objects solid ("yes") or not solid ("no").

statobj_solid_id   48   "yes"
statobj_solid_id   43   "no"

Disable Pick-Up (_nobonus_)
Make a pick-up item (map id number) non-pickable. Below's example won't let you collect the medikit (map id = 48) anymore.

statobj_nobonus_id   48

Billboard Objects (_board_)
Turns objects into flat directional ones.

Options are: vert_back, vert_front, vert_mid, horiz_back, horiz_front, horiz_mid

statobj_board_id   48   "vert_mid"
statobj_board_id   37   "horiz_front"

Lit Objects (_lit_)
Make objects bright ("yes") or not ("no"). Requires shading to be enabled to have any effect.

statobj_lit_id   48   "yes"
statobj_lit_id   37   "no"

============================== ITEMS ==============================

A simple method to modify existing items. Another way is to use the Custom Objects method to create new items from scratch.

Names
cross = cross, treasure item
chalice = golden chalice, can be sold at auctions
chest = full of fake diamonds
crown = be the King of Wolfenstein
ammoclip = ammo clip placed on map
ammoclip2 = ammo dropped by enemies
ammobox = box full of little friends running faster than you
dogfood = don't steal the food man's best friend
food = dirty food on the floor
medikit = it's good for you
life = usually a blue sphere with a face on it
gascan = ammo for the flamethrower
rockets = missiles for the missile launcher or whatever you made out of it

Effects
_damage = damage to the player
_health = health for the player
_armor = armor for extra defense
_ammo = ammo amount
_gas = flamethrower ammo
_rockets = missile amount
_time = adds time to the countdown timer if enabled (in seconds)
_ignore "health" = you can't pick it up if you have maximum health
_ignore "armor" = you can't pick it up if you have maximum armor
_ignore "ammo" = you can't pick it up if you have maximum ammo
_ignore "gas" = you can't pick it up if you have maximum gas
_ignore "rockets" = you can't pick it up if you have maximum rockets

Some examples:

chest_time 10
ammobox_time 10
 Collecting the chest or ammobox item gives you 10 extra seconds for the countdown timer.

food_ignore "ammo"
 You can't pick up food if your ammo is at max.

cross_damage 10
cross_rockets 6
 This gives you 10 damage and 6 rockets when picking up the cross.

medikit_ammo 10
medikit_damage 50
medikit_rockets 42
 Collect a medikit and get 10 bullets, 42 rockets and 50 damage.

chest_armor 40
 Collect a chest and get 40 armor.

============================== CUSTOM OBJECTS ==============================

Method to create new sprite objects by using another sub-config file system. First define the file and place it inside the conf folder.

sprites   "spr.txt"

In SPR.TXT:

Assign Sprites (_num)
You assign a sprite (sprite number) to a map id (plane 2). With this you can place, for example, the object 54 in a map editor and it will spawn sprite 108.

spr_num   457   364
spr_num   4   77
spr_num   54   108

Sprite Animation (_frames, _animspeed)
Animates the static object with a given speed and makes it a little bit less static. For example id 54 could have 11 frames.

spr_frames     54   11
spr_animspeed   54   3

Sprite State (_state)
Makes a sprite block (solid) or not (clip).

spr_state   54   "solid"
spr_state   94   "clip"

Sprite Features (_feature)
Hate that a sprite keeps spinning? Make it a directional sprite (billboard in abcDoom lingo) instead. A flat cardboard cutout that can be either centered or at the edge of a tile. They are often used to create windows and fences in mods. Make a sprite translucent with the dither option for a glass wall or a ghostly effect. The bright option makes a sprite lit if light shading is enabled.

  • vert_front
  • vert_back
  • vert_mid
  • horiz_front
  • horiz_back
  • dither
  • bright

spr_feature   54   "horiz_front"
spr_feature   94   "vert_mid"
spr_feature   94   "dither"
spr_feature   66   "bright"
spr_feature   23   "vert_back"

Projectile Pass (_projectiles)
Lets a sprite pass (pass) or not (nopass) projectiles. In other words lets your rocket fly through a column or not.

spr_projectiles   54   "nopass"
spr_projectiles   94   "pass"

Sprite Use (_use)
Gives a sprite a deeper meaning than to just stand around and look pretty. It could be an item to pick up or be a log file.

  • treasure:n: Gives you n-times treasure.
  • ammo:n: Gun food.
  • gas:n: Flamethrower gasoline.
  • rocket:n: Gives you n-amount of rockets.
  • weapon:n: Gives you a weapon of choice (0-5)
  • health:n: Make it a medikit.
  • armor:n: Or a piece of fancy body armor.
  • damage:n: It could be a trap or poisoned food.
  • key:n: n can be 1 or 2 cause there are only 2 keys.
  • time:n: Time for timed missions. See Countdown Timer in Various Things
  • xp:n: Gain experience points to level up.
  • lamp: Makes the sprite fully bright if shading is enable.
  • push: Makes it pushable by pressing the use key. Perhaps for some puzzles.
  • carry: Carry the sprite around. Maybe to block a way for the enemies?
  • log:yourfile.xxx: Use key will bring up a text screen (see Readme/Endart) using the specified file.

spr_use   34   "ammo:25"
spr_use   46   "rocket:1"
spr_use   95   "gas:111"
spr_use   24   "weapon:4"
spr_use   24   "health:69"
spr_use   25   "damage:51"
spr_use   26   "key:1"
spr_use   27   "key:2"
spr_use   28   "time:240"
spr_use   29   "armor:100"
spr_use   432   "xp:301"
spr_use   433   "lamp"
spr_use   434   "push"
spr_use   435   "carry"
spr_use   436   "log:conf/story/HELPART.WLt"

hud_x id amount
hud_y id amount
hud_align id alignment ("bottom_right", "bottom_left", "bottom_center", "top_right", "top_left", "top_center")

1 = statusbar
2 = area
3 = items
4 = items total
5 = secrets
6 = secrets total
7 = score
8 = kills
9 = enemies
10 = face
11 = lives
12 = health
13 = ammo combined
14 = ammo gas
15 = ammo rockets
16 = ammo bullets
17 = key gold
18 = key silver
19 = timer
20 = armor
21 = stamina

hud_x 1 1111
hud_y 1 1111
Disables the main statusbar.

hud_y 9 75

Adds enemy count and moves it to the top right.

hud_x 17 11
hud_y 17 33

Moves the gold key to the bottom left.

============================== XP System ==============================

The key to bring up the player status screen can be found in the control menu.
XP values are reset when a new game is started.
The msg_level command could be abused for plain story-telling. Player stats stay 0 and only 5 specific items in a level give points for a message to pop up. 1 item collected, story screen, 5 items collected, next story screen and so on.

First you define the sub-config file anywhere in GAMEDICT.WL6.

xplevels   "xp.txt"

In XP.TXT:

Word Definitions
For the level-up and status window.

str_stamina   "endurance"
str_hp     "hitpoints"
str_defense   "def"
str_armor   "armor"
str_speed   "speedometer"
str_strength   "force"
str_ammo   "mana"
str_rocket   "magic"
str_gas     "ether"
str_xp     "wisdom"

Level Points
The amount of experience points you need for each level. You start at level 0 and need 100 points to level 1 and so on. Points can be either gained by killing enemies or collecting items.

xptolevel   0   100
xptolevel   1   200
xptolevel   2   350
xptolevel   3   500
xptolevel   4   1000
xptolevel   5   2500

Level Stats (stat level amount)
The amount the player stats increase with each level.

max_damage   0   10
max_damage   1   16
max_damage   2   23
Additional damage to your weapon damage. If your weapon does 200 damage the overall damage will be 216 on level 1.

max_health   0   150
max_health   2   170
max_health   3   200

max_stamina   0   200
max_defense   0   10
max_armor   0   10
max_ammo   0   20
max_gas     0   5
max_rockets   0   1
Max value for each level. Max health, max ammo, ... take priority over values in the mod config with the exception of max_speed.

max_speed   0   80
max_speed   2   120
movement_speed from the mod config has a higher priority. For possible case of ice-skating levels.

Stamina
The amount the player stats increase with each level.

stamina_refresh   0   1
stamina_refresh   8   2
stamina_cooling   0   120
stamina_cooling   1   100
The amount stamina regenerates on level 0 while not actively using it. Cooling is the amount of time (in tics) the player has to wait after draining all stamina.

stamina_cost   0   1   // knife
stamina_cost   1   2   // pistol
stamina_cost   2   3   // mg
stamina_cost   4   120   // flamethrower
stamina_cost   5   100   // rocketlauncher
stamina_cost   7   10   // running
Amount of stamina drain caused by each weapon (0-5) and running (7). There are no levels in use here. Just the universal cost of each.

stamina_color   255
Bar color for the Statusbar.

Level Message & Colors
The amount the player stats increase with each level.

msg_color   155
Font color for the status and level-up screen. Look up the palette index for the color.

msgbox_color   255
Background color for the status and level-up screen. The value 255 will remove the background for the level-up.

msg_level   1   "you leveled up and got +1 iq"
msg_level   2   "you leveled up and got +2 iq"
msg_level   3   "You have turned into the legendary Wolf Slayer and have aquired \nthe power to rip and tile. Health Stat is up by 100."
Message that displays when levelling up. Long messages might require newlines to be put manually by adding \n.

msg_default   "As you conquered the vast Waste Lands and, \nwe crushed the enemies before you\nthe sky darkened. The gates of hell opened\nand a vile creatures greets you.\nA devilish djinn curses you with new powers,\nand vanishes into thin air a moment later.\nYou feel your blood pulsating with an unknown life force.\nYour muscles put even Hercules to shame and reached a state of infinite stamina."
The default level-up message if no specific one is set for a level. Level 1-3 will display above message and then it will use the default one for any further level. The default one automatically counts and displays the stats at the bottom of the message.
